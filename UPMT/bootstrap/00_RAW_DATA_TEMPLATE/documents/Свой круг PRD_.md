# **PRD — Product Requirements Document**
## **Проект: «СВОЙ КРУГ»**
### **Версия: 1.0**
### **Подготовлено: стратегический ассистент проекта**

## **1. Обзор проекта**
### **1.1. Название**
**«СВОЙ КРУГ»** — экосистема женских бизнесов и клиентов, объединённая цифровым клубом, который усиливает взаимодействие между участницами и создаёт добавочную ценность внутри сети.

### **1.2. Краткое описание**
«Свой Круг» — мобильное приложение и клубная платформа, объединяющая несколько независимых бизнесов (гастромаркет, салон красоты, стоматология, флористика, оптика, клиника, юридическая фирма и др.) в единую систему.
 Каждый клиент одного из бизнесов становится участницей клуба, получает доступ к единому профилю, бонусам, статусам и привилегиям за активность в экосистеме.

### **1.3. Цель проекта**
Создать цифровую экосистему женских бизнесов, способствующую росту клиентской вовлечённости, увеличению LTV (Lifetime Value) и формированию ценностного сообщества вокруг брендов.

## **2. Целевая аудитория (User Personas)**
### **2.1. Основная аудитория**
Женщины 25–50 лет, платёжеспособные, живущие в городах-миллионниках.

Ценят комфорт, эстетику, качественные услуги и персональный подход.

Часто посещают салоны красоты, клиники, флористику, гастромаркеты.

Активны в Instagram*, Telegram, VK.

### **2.2. Вторичная аудитория**
Собственницы женских бизнесов.

Маркетологи, управляющие, администраторы точек продаж.


## **3. Проблематика и возможности**
### **3.1. Проблемы**
У каждого бизнеса своя клиентская база и IT-система.

Нет общего пространства и кросс-продаж.

Потеря данных и отсутствие общего анализа клиентского поведения.

Недостаточная лояльность и возвратность клиенток.

### **3.2. Возможности**
Создать общий цифровой слой над существующими IT-системами.

Использовать единую CRM и историю покупок для бонусов и рекомендаций.

Повысить клиентскую вовлечённость через статусное членство.

Сформировать уникальное женское комьюнити с высокой эмоциональной привязанностью.


## **4. Общее видение продукта**
### **4.1. Формат**
Мобильное приложение для клиенток.

Веб-админка для собственниц бизнеса.

Централизованная база данных (аналитика, интеграции, кросс-маркетинг).

### **4.2. Принцип работы**
Клиент совершает покупку в одном из бизнесов.

Данные о покупке автоматически передаются в систему «Свой Круг».

Участнице начисляются баллы / статусные очки.

Приложение показывает историю активности, статус, бонусы и предложения партнёров.

Собственницы видят аналитику по клиентам и эффективность коллабораций.

## **6. Техническое видение**
### **6.1. Архитектура**
**Frontend:** React Native / Flutter (мобильное приложение).

**Backend:** FastAPI (Python) / Node.js.

**Database:** PostgreSQL + ClickHouse (аналитика).

**Integrations:** REST API / GraphQL / 1С connector.

**Infrastructure:** Docker + Kubernetes, облачный сервер (РФ).

### **6.2. Безопасность**
Хранение данных в РФ (152-ФЗ).

Шифрование персональных данных (AES-256).

Авторизация по токенам JWT.

Ограничение доступа по ролям (RBAC).


## **7. UX/UI Концепция**
### **7.1. Визуальный стиль**
Основные цвета: **Tiffany (#81D8D0)**, **молочный беж**, **жемчужный белый**.

Акценты: полупрозрачные элементы, **Liquid Glass эффект**, лёгкие градиенты.

Типографика: современный геометрический шрифт с кириллической поддержкой.

### **7.2. Принципы UX**
Минимум кликов, максимум эмоций.

Геймифицированный прогресс (кольцо достижений).

Визуальные бейджи статуса.

Прозрачная и простая навигация.


## **8. MVP (Минимально жизнеспособный продукт)**
**Состав MVP (3 месяца разработки):**
Мобильное приложение с регистрацией, личным кабинетом и бонусной системой.

API-интеграция с CRM/1С для одного бизнеса.

Простейшая админ-панель (настройка акций, выгрузки, отчёты).

Аналитика и история покупок.

**MVP не включает:**
Сообщество и ивенты.

Полноценную AI-персонализацию.

Мультибизнес-аналитику (выйдет на этапе 2.0).

## **12. Рекомендации**
Сосредоточиться на **простой, но красивой версии MVP**.

Максимум пользы при минимальных изменениях в инфраструктуре партнёров.

Создать единый коммуникационный центр (Telegram + CRM).

Формировать клуб как **ценностное пространство, а не просто бонусную систему**.
