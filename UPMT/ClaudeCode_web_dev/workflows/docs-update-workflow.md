# 📚 DOCS UPDATE WORKFLOW

**Версия:** 1.0.0

## Шаги

1. **Identify Outdated:** Найди устаревшие docs
2. **Update Content:** Обнови docs согласно текущему code
3. **Cross-Reference:** Проверь ссылки между docs
4. **Commit:** git commit -m "docs(module): update description"

## Триггеры (Project Rules)

Docs обновляются автоматически при:
- Изменении API endpoints
- Добавлении новых модулей
- Изменении архитектуры
- Принятии архитектурных решений
