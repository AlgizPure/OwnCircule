### Context Switching

**Severity:** High  
**Frequency:** Daily (multiple times)  
**Affected Users:** Team Leads, Developers  
**Source:** User interviews (8/10 mentioned)

**Description:**
Users need to switch between multiple tools (Jira, Slack, Email, Docs) to complete a single task. This breaks focus and wastes time.

**User Quotes:**
> "I spend more time switching tools than actually working"
> "I lose track of where information is"

**Impact:**
- 30% of time spent context switching
- Increased frustration
- Reduced productivity

**Potential Solutions:**
- Integrate tools into one interface
- In-context discussions
- Unified search

**Status:** In Progress (v2.0)

---

### Unclear Requirements

**Severity:** High  
**Frequency:** Weekly  
**Affected Users:** Developers

**Description:**
Task descriptions are vague or incomplete. Developers must hunt for requirements across Slack, meetings notes, and docs.

**User Quotes:**
> "Requirements are scattered everywhere"
> "I spend hours just figuring out what to build"

**Impact:**
- Wasted development time
- Incorrect implementations
- Back-and-forth with PMs

**Potential Solutions:**
- Requirement templates
- Checklist for completeness
- Link related docs
- Inline clarifications

**Status:** Planned (v2.1)

---

### Slow PR Reviews

**Severity:** Medium  
**Frequency:** Every PR  
**Affected Users:** Developers

**Description:**
PRs sit for days without review. Developers are blocked, context is lost.

**User Quotes:**
> "I forget what the PR was about by the time it's reviewed"
> "I'm always blocked on reviews"

**Impact:**
- Delayed features
- Context loss
- Developer frustration

**Potential Solutions:**
- PR review notifications
- Review assignment automation
- Review time SLA

**Status:** Solved (v1.5)
