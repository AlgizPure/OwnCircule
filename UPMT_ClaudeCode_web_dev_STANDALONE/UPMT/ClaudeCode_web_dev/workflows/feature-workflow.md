# 🔄 FEATURE DEVELOPMENT WORKFLOW

**Версия:** 1.0.0

## Шаги

1. **Read Requirements:** Прочитай docs/requirements/[module].md
2. **Plan Implementation:** Создай технический план, покажи пользователю
3. **Write Code:** Пиши код по плану, следуй Tech Stack
4. **Update Docs:** Обнови docs если нужно (Project Rules!)
5. **Test:** Manual или automated тесты
6. **Commit:** git commit -m "feat(module): feature description"
7. **Checkpoint:** Сохрани checkpoint в .dev/checkpoints/
8. **Report:** Добавь iteration report в session log
9. **Progress:** Обнови modules_status.md и sprint_current.md

## Code Quality

- TypeScript strict mode
- Error handling
- Comments для сложного кода
- Tests перед commit

**См. также:** dev-orchestrator.md для полного цикла
